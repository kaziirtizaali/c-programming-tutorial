4.1: Is there more than one null statement ?

Ans: Sort of. You can use '';'', ''0;'', or ''1;'' - they will all act like a null statement. Only the first is a ''true'' null statement (all bits zero). They are basically equivalent. Note that (void *) 0; is a null statement of type pointer to void, for instance.

 

4.2: Is { } a null statement ?

Ans: No. { statement-list[opt] } is a compound statement. An empty block is not the same as a null statement, however, although it can be used in many of the same places. It's really a null block.

 

4.3: I use the statement #define NULLSTMT(F) (F) ; to allow me to cast a null statement to an appropriate type.

Ans: This trick, though popular in some circles, does not buy much. The resulting code is invalid, and will not compile. This (in the author's opinion) outweighs any arguable type consistency. It may be more common in industrial code. If it becomes common practice, C++ will probably legalize it. 

 

4.4: Is a null statement a null pointer ?

Ans: No. A null pointer is a pointer where all of the address bits are zero (no matter what the segment bits are), and can be obtained by typing '(char *) (int) 0'. A null statement is not a pointer to anything. They are not interchangeable, although you can combine them to get an effectively-null statement, such as NULL;. This does not buy you anything.

 

4.5:  What is a null pointer ?

Ans: There are times when it’s necessary to have a pointer that doesn’t point to anything. The macro NULL, defined in , has a value that’s guaranteed to be different from any valid pointer. NULL is a literal zero, possibly cast to void* or char*.
